# Coclaw 架构设计文档

## 概述

Coclaw 是一个基于 OpenClaw 的局域网 AI 协作工具，采用模块化、可扩展的架构设计。本文档详细描述了 Coclaw 的系统架构、组件设计和技术决策。

### 核心组件

1. **CLI 工具**: 命令行界面，提供用户交互
2. **服务器集群**: 处理网络通信和消息路由
3. **Agent 管理器**: 管理 OpenClaw Agent 生命周期
4. **文件传输系统**: 处理文件上传和下载
5. **权限管理系统**: 控制 Agent 间访问权限
6. **服务发现模块**: 自动发现局域网中的服务器

## 技术栈

### 后端技术

- **运行时**: Node.js (v14+)
- **CLI 框架**: Commander.js
- **Web 框架**: Express.js
- **WebSocket**: ws 库
- **进程管理**: child_process
- **文件操作**: fs-extra
- **用户交互**: inquirer
- **配置管理**: 基于 JSON 的自定义配置系统
- **调试工具**: debug 库
- **HTTP 客户端**: 内置 http/https 模块
- **文件上传**: multer 中间件
- **数据格式**: YAML 解析器

### 网络协议

- **主协议**: WebSocket (实时通信)
- **辅助协议**: HTTP (文件传输、API)
- **发现协议**: UDP 广播/mDNS
- **安全协议**: TLS/SSL (可选)

### 数据存储

- **配置存储**: JSON 文件 (`~/.coclaw/`)
- **临时文件**: 内存缓存 + 磁盘缓存
- **状态管理**: 内存数据结构 + 定期持久化

## 技术决策记录

### 1. 选择 Node.js 的原因

- 异步 I/O 适合高并发场景
- 丰富的生态系统
- 易于开发和部署
- 良好的跨平台支持

### 2. 选择 WebSocket 的原因

- 低延迟双向通信
- 适合实时应用场景
- 浏览器原生支持
- 良好的工具生态

### 3. 选择 JSON 文件存储的原因

- 简单易用
- 易于调试
- 无需外部依赖
- 适合小规模数据

### 4. 选择 Commander.js 的原因

- 成熟的 CLI 框架
- 丰富的功能支持
- 活跃的维护
- 良好的文档

## 性能指标目标

### 1. 延迟目标

- 消息传输延迟: < 100ms (局域网)
- 文件传输延迟: 取决于文件大小
- 命令响应时间: < 1s
- 服务器启动时间: < 5s

### 2. 吞吐量目标

- 并发连接数: 100+
- 消息处理速率: 1000+/s
- 文件传输速率: 100MB/s (局域网)
- 内存使用: < 512MB (基础)

### 3. 可用性目标

- 系统可用性: 99.9%
- 数据持久性: 99.99%
- 故障恢复时间: < 30s
- 备份恢复时间: < 5min

## 总结

Coclaw 采用模块化、可扩展的架构设计，注重性能、安全和易用性。通过清晰的组件边界和良好的接口设计，系统能够适应不同规模的使用场景，并为未来的功能扩展预留了空间。
